VelocityClassBasedFileMaker    Property List     v1.6.0

In the header of the main templates for input artefacts of this plugin the following artefact properties can be defined:

1.   TargetDir Folder in your file system where to write the generated content. If not defined, output will only be written in the plugin’s output directory. Use “<applicationRootDir>” (without quotes) to define the directory where the start script startMOGLiCodeCreator is located.

2.   TargetFileName Name of file in which the generated content is saved.

3.   CreateNew If “true” both the target folder is created if not existing and the target file is overwritten if existing. Default is false.

4.   NameOfValidModel Determines that the target file is only modified if the name of the model provided by the StandardModelProvider is equal to the model name defined here. If not provided, the template is applied to all models.

5.   OutputEncodingFormat Encoding applied when generating the artefact’s output file(s). Default is UTF-8. Since 1.1.0

6.   SkipGeneration If “true”, code supposed to be created is not generated. Default is false. Used to suppress generation of files for certain classes, e.g. @SkipClass NOT $classDescriptor.getMetaInfoValueFor("databaseRelevant"). Since 1.2.0

7.   ReplaceToNumberSign Any string that will be replaced by the # symbol after Velocity has merged model and template and before the output file is created. Since Velocity interprets all number signs and its following chars as comment, these comments are removed during the Velocity merge of model and template. Thus, no number sign can get through this merge. As a workaround to get number sign in the final output, a number sign placeholder can be defined by this artefact property. Since 1.6.0
